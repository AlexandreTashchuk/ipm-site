Bad Interface Design Example: Clip

My choice for a bad interface example was clip because of a few reasons:

  • Whenever I want to login to clip, it tells me that there are inquiries ongoing even though I already filled them and, after clicking the "proceed button", 60% of the time it takes me back to the login menu:

    • It then takes me back to this page:

    Which then takes me back to the inquiry page. A proposition to fix this would be to remove the menu informing the user that there are inquiries ongoing to the users that already filled them out since, after logging in, the user has the option to check them;

  • Another issue with CLIP is that user sessions expire very quickly, and there is no option to adjust the login duration. Combined with the first problem, this makes the system especially time-consuming to use. An easy fix for this would be to add an option to adjust the session duration, or at least extend it in the first place;

  • A major problem with CLIP is its poorly optimized database design, which causes many actions on the website to take a long time to process. For this issue there is no easy fix, since the database would need to be completely overhauled but, if it were, it would save a lot of the users' time;

  • Finally, some of the button labels in CLIP are confusing. For example, when trying to pay tuition, it is not immediately clear which option to select, so a lot of the times, I end up pressing the wrong one.

    • A simple fix for this would be to change the button labels to something more descriptive, like "Pagar propinas" instead of just "Dados para pagamento" and "Histórico de pagamentos" instead of just "Propinas".
